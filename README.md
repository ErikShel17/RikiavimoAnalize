# \*Projektas atliktas studijų tikslais.\*

# \# Rikiavimo algoritmų lyginamoji analizė

# 

# Šiame projekte atliekama įterpimo (\*Insertion Sort\*) ir sąlajinio (\*Merge Sort\*) rikiavimo algoritmų lyginamoji analizė. Programa realizuota C++ kalba, siekiant praktiškai patikrinti algoritmų efektyvumą esant skirtingiems duomenų rinkinių dydžiams bei pradinėms būsenoms.

# 

# \## Projekto aprašymas

# Projektas skirtas įvertinti dviejų skirtingo sudėtingumo klasių algoritmų veikimą:

# \* \*\*Insertion Sort\*\* – $O(n^2)$ sudėtingumo algoritmas, tinkantis mažiems arba jau iš dalies surikiuotiems masyvams.

# \* \*\*Merge Sort\*\* – $O(n \\log n)$ sudėtingumo algoritmas, pasižymintis stabilumu ir efektyvumu dirbant su dideliais duomenų kiekiais.

# 

# Eksperimento metu matuojamas vykdymo laikas (mikrosekundėmis), atliekamų palyginimų bei elementų sukeitimų skaičius.

# 

# \## Funkcionalumas

# \* Duomenų nuskaitymas iš failų.

# \* Rikiavimo procesų vykdymo laiko matavimas.

# \* Palyginimų ir sukeitimų operacijų skaičiavimas.

# \* Rezultatų analizė su trimis skirtingais duomenų tipais:

# &#x20;   \* Nesurikiuoti duomenys.

# &#x20;   \* Atvirkščiai surikiuoti duomenys.

# &#x20;   \* Jau surikiuoti duomenys.

# 

# \## Reikalavimai paleidimui

# \* \*\*Kompiliatorius:\*\* C++11 arba naujesnis standartas.

# \* \*\*Aplinka:\*\* „Visual Studio“ (projektas turi `.sln` failą) arba bet kuri kita C++ aplinka.

# 

# \## Kaip paleisti

# 1\. Atsisiųskite arba klonuokite šią saugyklą.

# 2\. Atidarykite `RikiavimoAnalize.sln` failą su „Visual Studio“.

# 3\. Užtikrinkite, kad programai reikalingi duomenų failai yra šakniniame aplanke.

# 4\. Sukompiliuokite ir paleiskite programą (\*Run\*).

# 

# \## Eksperimento rezultatai

# Tyrimas patvirtino teorinius algoritmų sudėtingumo rodiklius. Sąlajinis rikiavimas išlieka pranašesnis dirbant su didelės apimties nesurikiuotais duomenimis, tuo tarpu įterpimo algoritmas demonstruoja geriausius rezultatus tik su jau surikiuota seka.
